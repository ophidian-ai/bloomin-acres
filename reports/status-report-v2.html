What Changed
Updates since Version 3.0 (March 7, 2026)

Performance Optimization

Tailwind CDN Replaced with Compiled Build
Removed ~350 KB Tailwind CDN script from all 6 pages. Replaced with a compiled, minified CSS file containing only used classes. Result: 5.5 KB total CSS. Tailwind CLI builds automatically on Vercel deploy via npm run build.
Build Pipeline Added
Introduced Tailwind CLI build step via package.json and vercel.json buildCommand. Generates optimized CSS at deploy time. Foundation for future JS bundling and minification.

Code Quality

Shared Utility Module (js/utils.js)
Extracted showToast(), escHtml(), and formatPrice() from 5 HTML files into a single shared module. Standardized toast element IDs and timeout values across all pages.
CSP Tightened
Removed Tailwind CDN from Content-Security-Policy whitelist (no longer needed). Tighter script-src and style-src directives.

Bug Fixes

Production API Crash Fix (ALLOWED_ORIGIN)
All Vercel serverless functions were returning 500 due to a trailing newline in the ALLOWED_ORIGIN environment variable. Node.js rejects newlines in HTTP header values. Added .trim() to all 5 API endpoints as a defensive measure.

Code Quality Assessment

Strengths (Score: 7.5/10)

  • Clean Supabase integration -- RLS policies on all tables, proper auth flow, parameterized queries
  • Stripe best practices -- Webhook signature verification, idempotent order creation, PCI-compliant checkout
  • Admin/customer separation -- Role-based UI with admin-only sections properly gated
  • CMS-driven content -- Landing page and menu fully editable without code changes
  • Modular API layer -- Separate files for checkout, subscribe, portal, products, webhook
  • XSS prevention -- escHtml() used on user-facing product names and descriptions
  • Stock validation -- Dual client + server validation prevents overselling at checkout
  • Security headers -- CSP, HSTS, X-Frame-Options, nosniff, Referrer-Policy configured
  • Shared utilities -- showToast, escHtml, formatPrice centralized in js/utils.js
  • Build pipeline -- Tailwind CLI compiles CSS automatically on deploy via vercel.json buildCommand

Areas for Improvement

  • Massive inline scripts -- account.html has ~1,200 lines of inline JS. Not cacheable, not testable, blocks parsing.
  • Build pipeline (basic) -- Tailwind CLI compiles CSS. JS still served raw. Future: add esbuild for bundling and minification.
  • Variable naming inconsistency -- sb vs supabase, currentUser vs session.user across pages.
  • Magic numbers -- Toast timeout now standardized (3000ms) but other constants still scattered.
  • No error boundaries -- Network failures in product/menu loads show no fallback UI.
  • Duplicated fetch patterns -- /api/config fetched on every page without session caching.
